# Проект

Проект "Космический корабль". 1961 год, планируется запуск корабля "Восток" с Юрием Гагариным.
Твоя задача проверить работоспособность корабля.
Если найдешь неисправности, нужно будет быстро починить! Космическая гонка идет во всю!
ТЕСТИРОВАТЬ НУЖНО ТОЛЬКО ПУБЛИЧНЫЕ МЕТОДЫ!

Этап №1 Проверка каждого модуля (класса) отдельно (unit test).
+ 1. Сделать форк от репозитория https://github.com/Ablazzing/SpaceShuttle
+ 2. Изучить проект: Корабль состоит из ракеты носителя (rocket) и капсулы (capsule).
+ 3. Ракета (класс Rocket) носитель состоит из 3 ступеней двигателей (класс Engine).
Нужно создать тест, который проверяет, что у любой созданной ракеты стоят 3 двигателя
(т.е. не равны null).
Если окажется, что проблема с двигателями есть, то поменять код в конструкторе Rocket.

+ 4. У двигателей есть предел количества топлива (100_000 литров).
Написать тест, который проверяет наличие ошибки LimitFuelException при создании двигателя с количеством топлива 200_000.
Написать тест, который проверяет наличие ошибки LimitFuelException при дозаправке двигателя, если топлива больше лимита.
Если ошибки не появятся, то поменять код в Engine.

5. За запуск двигателя отвечает метод start.
Нужно написать тест, который будет ожидать NotEnoughFuelException, если у двигателя меньше 60_000 литров топлива.
Если ошибки не будет, то нужно переписать метод start.

6. Метод run Ракеты запускает двигатели по очереди, разгоняя корабль и вынося его на орбиту.
Написать тест, который проверит успешный запуск при отсутствии ошибок из двигателей.
Двигатели должны быть заглушками, т.е. не привязаны к конкретной реализации.

Написать тест, который проверит наличие ошибки при старте,
если 1 из двигателей заглушек, выкидывает ошибку.

7. Теперь поработаем с капсулой (capsule).
Так как корабль пилотируемый, то внутри него есть капсула с космонавтом.
Каждый космонавт, должен быть здоров перед полетом
(космический корабль не должен взлететь, если космонавт не здоров).
Поэтому напиши два теста:
первый тест должен проверить, что корабль полетит, если космонавт здоров;
второй тест будет ожидать ошибку CosmonautIsNotReadyException, если космонавт болен.
Космонавт должен передаваться в виде заглушки.

Этап №2 Проверка работоспособности группы модулей (integration test).
В тестах ниже не будет заглушек.
1. При создании капсулы, внутри нее создается система жизнеобеспечения.
Нам нужно проверить, что никто не забыл положить баллон с кислородом в систему жизнеобеспечения.
Напишите тест, который:
1. Создает капсулу
2. Проверяет что полученная из капсулы системы жизнеобеспечения не пуста
3. Проверяет что у системы жизнеобеспечения не пустой кислородный баллон.
Если код не проходит тест, поправить код.

2. Когда мы сажаем здорового космонавта в космический корабль, проходит мало времени,
 чтобы он успел заболеть.
Напишите тест, проверяющий что при посадке в корабль, космонавт не заболевает.
Нужно создать космический корабль, и посадить туда здорового космонавта.
После этого проверить его здоровье - оно не должно поменяться.
Если код не проходит тест, поправить код.

3. После запуска ракеты, нужно проверить, что у всех двигателей количество топлива стало 0.
Сделай на это тест.

4. Инженеры конструкторы передали список расчетов (rocket_data.csv в тестовых ресурсах).
Необходимо создать тест, который берет из файла параметры для создания ракеты и запустить ракету:
если result для эксперимента равен true, то ошибок не будет при создании и запуске не будет.
Если false то метод должен завершиться с ошибкой.
